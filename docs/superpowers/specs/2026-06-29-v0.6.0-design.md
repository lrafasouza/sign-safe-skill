# sign-safe v0.6.0 — Design Spec

> Status: approved in brainstorming 2026-06-29. Next: `/grill-me` → writing-plans → gated implementation.
> Baseline: v0.5.1 (`main @ 910b089`) — 777 tests / 40 files, fail-closed, dual-SDK cross-validation, MCP, `--simulate`, 37/37 synthetic attack replay, 100 benign mainnet corpus, ~14-program clear-signing registry.

## Goal

Make sign-safe *demonstrably* stronger and more adoptable **without diluting the honesty discipline**: real on-chain attack evidence, installable via npm, deeper IDL-verified clear-signing coverage, and *comprovable* test robustness.

## Invariant principles (apply to every workstream)

- No overclaim, no fabrication. Every recall/precision metric carries the curated-corpus caveat.
- Only real outputs (verdicts/metrics generated by the real CLI/tests, never hand-authored).
- `verify:all` green before any commit; `skill/src/banned.ts` respected; locked docs stay green.
- Each workstream is an isolated unit (own files, clear interface, testable alone) and lands behind the same gated process used for v0.5.1: implement (Sonnet) → independent Opus review → fix-loop → commit only if green.

## Locked decisions (from brainstorming)

- **Scope:** A + B + C(scoped) + D + E + F + CLI GIF. Deferred: G (hosted web demo), H (devnet+Explorer), I (lazy-load doc split).
- **B (npm):** I prepare 100% (prepublish, npm README, dry-run green, `PUBLISH.md`); the user runs `npm publish`.
- **A (corpus bar):** only publicly *documented* incidents; each sample carries an Explorer link + source; the user approves every candidate before it enters the corpus.
- **C (registry):** Jupiter, Raydium, Orca, Kamino, Drift — per-instruction, every discriminator verified vs the canonical IDL.

## Workstreams

### A — Real on-chain malicious corpus (flagship)

**What:** capture real, publicly-documented attack/exploit transactions, decode them offline through the deterministic core, and prove HOLD/REJECT — each with provenance.

**Design:**
- New directory `skill/corpus/malicious/` mirroring `skill/corpus/benign/`. Each entry: `{ signature, slot, cluster, b64, meta: { source_url, incident, expected_verdict, captured_at } }`.
- Capture script `skill/corpus/capture-malicious.ts` (mirror of `capture-benign.ts`): fetch raw tx by signature. Source of bytes: the Helius MCP in-session (no user RPC required) or a provided RPC; the fixture is then frozen to disk.
- Replay test asserts each entry → HOLD or REJECT, **never SIGN**. Integrated into `verify:all` as a *separate* replay, clearly labeled "real documented incidents" vs the existing 37 synthetic.
- `docs/real-attacks.md` lists every sample with Explorer link + incident source + the verdict it produced.

**Curation bar (locked):** only incidents with a public postmortem/report; user approves each candidate. If a real tx yields HOLD (not REJECT), record HOLD honestly.

**Honesty constraint:** the corpus is "documented incidents that sign-safe holds/rejects," NOT "all attacks." Count is reported as-is.

**Success:** ≥ a small honest N of real incidents (target 5–10 to start) replayed, each with provenance, 0 false SIGN, CI green. Headline becomes "N real documented attacks held/rejected" alongside the 37 synthetic.

**Depends on:** tx bytes (Helius MCP / RPC) + user approval of the incident list.

### B — npm publish (prepared; user runs)

**What:** make sign-safe installable so an evaluator or agent can use it without cloning (`npx <name>`, `sign-safe-mcp`).

**Design:** finalize `package.json` (`bin` already present), audit `files[]`, ensure `prepublishOnly` builds, write a focused **npm README** (the Evaluator Quickstart, not the 800-line one — npm renders the README), review `npm pack --dry-run` contents, bump to 0.6.0, and write `PUBLISH.md` with the exact `npm publish` command + a dry-run transcript.

**Open decision (→ grill):** `npx sign-safe` requires the package name to be `sign-safe` (today it is `sign-safe-skill`). Check availability; recommend claiming `sign-safe` (fallback `sign-safe-skill` → `npx sign-safe-skill`, or `@scope/sign-safe`).

**Success:** `npm pack --dry-run` clean, README renders, version 0.6.0, publish command + transcript ready. User runs publish.

**Depends on:** npm name availability (I check); user runs the command.

### C — Registry expansion, IDL-verified

**What:** add per-instruction danger classification for Jupiter, Raydium, Orca, Kamino, Drift to the clear-signing registry.

**Design:** extend `skill/catalog/program-registry.json` with each program's dangerous/safe instruction discriminators. **Every discriminator verified against the canonical IDL** (Anchor: `sha256("global:<ix_name>")[0:8]`), recording the source IDL reference. A verification test recomputes discriminators from the recorded instruction names (mirrors the existing 133 Anchor discriminator checks) so they cannot drift. Classify custody-risk instructions (withdraw, set authority, update admin, proceeds redirection) as dangerous → HOLD/REJECT; routine swaps as recognized.

**Honesty constraint:** never guess a discriminator. If an IDL is not obtainable/verifiable, the program stays recognize-only and the gap is documented.

**Success:** 5 protocols added, every discriminator self-verified in tests, CI green, README registry count updated honestly (e.g., 14 → 19).

**Depends on:** IDLs (fetched via RPC / MCP / official repos).

### D — rpc-adversarial coverage completion

**What:** close the gap the v0.5.1 reviewer flagged — prove "RPC enrichment cannot remove or downgrade findings derived from the signed bytes" across **all** enrichment paths, not just the legacy fixture.

**Design:** add a v0+ALT fixture and a Token-2022 mint fixture that actually invoke the ALT-resolution and mint-extension fetchers (assert fetcher call-count > 0). Assert byte-derived findings survive adversarial/benign RPC responses, and that a forged ALT attempting to inject a hidden writable signer is caught fail-closed. Update the RPA comments so they match what is actually exercised.

**Success:** enrichment paths genuinely exercised, tests pass, comments accurate.

**Depends on:** nothing (offline; self-serve).

### E — Mutation testing + fuzz harness

**What:** prove the test suite has teeth (answer "are 777 tests real coverage or just a number?").

**Design:**
1. Mutation testing on the decode/classify/verdict core. Approach: **Stryker** (dev-only devDep — does not affect the zero-runtime-dependency *shipped* core) → `docs/mutation-report.md` with the real mutation score. (Fallback: a focused hand-rolled mutator if Stryker proves too heavy.)
2. Fuzz/property harness via the existing fast-check: mutate real + synthetic fixture bytes and assert the fail-closed invariant — mutating an authority/owner byte never yields SIGN.

**Honesty constraint:** report the real mutation score even if < 100%; document survivors.

**Success:** mutation score reported; fuzz invariant test green; CI green.

**Depends on:** nothing (self-serve).

### F — Grow benign corpus 100 → 500

**What:** more real benign mainnet txns → tighter, more credible precision / false-REJECT measurement.

**Design:** capture ~400 more recent benign mainnet txns (via Helius MCP / RPC) into a re-frozen slot set reaching 500; **recompute precision**; update `docs/precision-report.md` + README numbers honestly. Decision: **add** to reach 500 and re-freeze (keep provenance), recompute.

**Honesty constraint:** re-measure on the bigger set; report whatever the new HOLD rate / false-REJECT count is (no cherry-picking).

**Success:** 500 benign corpus, recomputed precision, docs updated, false-REJECT re-verified.

**Depends on:** tx bytes (Helius MCP / RPC).

### GIF — asciinema of the CLI

**What:** a short real terminal recording of `sign-review` on a REJECT (Squads hidden authority) and a SIGN, for the post/README.

**Design:** record a real CLI session (asciinema → GIF/SVG, or a faithful scripted capture if asciinema is unavailable). Real output only.

**Success:** a GIF/SVG asset in `sign-safe-post/` (and optionally embedded in README).

**Depends on:** asciinema (installable) — fallback faithful scripted capture.

## Out of scope (deferred, with reason)

- **G** hosted web demo — distribution/marketing, not core safety; adds hosting/maintenance.
- **H** devnet intercepted-vs-passed + Explorer — largely redundant with A (real corpus is stronger proof); needs Solana CLI (not installed).
- **I** lazy-load doc split — internal refactor, zero evaluator-visible payoff, risk to locked docs.
- Earlier cuts: address-poisoning / lookalike-recipient (needs reputation data), generic nested-CPI decoding (high effort/uncertain gain), wallet/IDE plugins (out of scope).
