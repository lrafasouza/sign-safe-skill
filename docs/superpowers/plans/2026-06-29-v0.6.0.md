# sign-safe v0.6.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Each workstream is gated: implement (Sonnet) → independent Opus review → fix-loop → commit only when the fast gate (`npm run build && npm test && npm run test:fixtures && npm run demo:attack-pack`) is green. The orchestrator runs the full `npm run verify:all` (incl. npm audit/pack) at the end. Never run `verify:all`/`npm audit`/`npm pack`/`npm ci` inside an agent (network → 180s watchdog kill).

**Goal:** Ship sign-safe v0.6.0 — real on-chain attack evidence, installable via npm as `sign-safe`, deeper IDL-verified clear-signing coverage, and comprovable test robustness — without diluting the honesty discipline.

**Architecture:** Each candidate is an isolated workstream (own files, clear interface). The deterministic offline core (`skill/src/*`, no network) stays pure; all network use stays in `skill/src/enrich.ts` and dev-only scripts under `skill/corpus/`. Real on-chain bytes are fetched once and frozen to JSON fixtures; tests read only frozen fixtures.

**Tech Stack:** TypeScript, vitest, fast-check, @solana/web3.js v1 + @solana/kit v2 (cross-validation), Stryker (dev-only mutation testing), Helius RPC (archival, for capture), tsx.

**Baseline:** branch `feat/v0.6.0` @ `910b089` (v0.5.1 merged): 777 tests / 40 files, 100 benign mainnet corpus, ~14-program registry, 37 synthetic attack replay.

**Honesty invariants (every task):** no overclaim; no fabrication (only real CLI/fetch outputs); every recall/precision metric carries the curated-corpus caveat; `banned.ts` respected; locked-doc test (`submission-assets.test.ts`) stays green; if a doc-locked string changes, update the assertion to the correct new value (never weaken it).

describe("fuzz: fail-closed under byte mutation", () => {

    const base = Buffer.from(readFixtureB64("02_setauthority_reject"), "base64");
